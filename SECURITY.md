# Security Policy

## Supported Versions

| Branch or version | Supported |
| --- | --- |
| `main` | Yes |
| Older branches, tags, and snapshots | No |

## Reporting a Vulnerability

Please report security vulnerabilities privately through GitHub Security Advisories:

`https://github.com/ymylive/lingma/security/advisories/new`

If private reporting is temporarily unavailable, contact the repository owner privately through GitHub before disclosing any details. Do not open a public issue for an unpatched vulnerability.

When reporting, include:

- Affected branch, commit, or deployment surface
- Reproduction steps and required prerequisites
- Expected impact, severity, and any proof-of-concept details
- Relevant logs or screenshots with secrets removed
- Suggested remediation if you already have one

## Response Targets

- Initial acknowledgement: within 3 business days
- Follow-up status update: within 14 calendar days
- Fix or mitigation target: within 30 calendar days when the issue is reproducible
