# 灵码（Lingma）— 交互式数据结构与算法学习平台

## 答辩稿

---

### 一、开场（约1分钟）

各位老师好，我今天答辩的项目是"灵码"——一个面向数据结构与算法学习的交互式 Web 平台。

学习数据结构与算法，传统方式往往是看书、看PPT、手写代码，学生很难直观理解算法的执行过程，练习反馈也不及时。灵码要解决的就是这个问题：把可视化演示、结构化教程、在线判题、AI 辅助出题和提示词工程训练整合到一个平台里，让学习更直观、练习更高效。

项目已部署上线，地址是 lingma.cornna.xyz。

---

### 二、技术架构（约2分钟）

灵码采用前后端分离的三层架构，通过 Docker Compose 编排部署。

**前端**使用 React 19 + TypeScript + Vite 构建，UI 层用 Tailwind CSS 做原子化样式，Framer Motion 做页面过渡和可视化动画。前端通过路由懒加载（React.lazy + Suspense）实现按需加载，首屏性能较好。支持中英文双语切换和深色/浅色主题。

**API 代理层**是一个 FastAPI 服务，负责：
- 转发前端请求到 AI 大模型接口（支持 OpenAI、DeepSeek、智谱、ModelScope 等多家供应商）
- 用户认证与注册（基于 SQLite 存储）
- 思维导图的云端同步存储
- 判题结果的 AI 复审（对用户提交的代码和判题结果做智能反馈）

**判题服务**是一个独立的 Node.js + Express 服务，支持 C、C++、Java、C#、Python 五种语言的在线编译运行和判题。容器以只读文件系统运行，tmpfs 挂载临时目录，drop 全部 Linux capabilities，限制 PID 数量，做了基本的沙箱安全隔离。

三个服务通过 Docker Compose 编排，Nginx 做反向代理统一入口，部署脚本基于 paramiko 实现一键自动化部署到 VPS。

### 三、核心功能演示（约5分钟）

#### 1. 算法可视化

平台内置了链表、栈、队列、二叉树、排序等多种数据结构的交互式可视化组件。以排序为例，支持冒泡、选择、插入、希尔、归并、快速、堆排序共 7 种算法的动画演示，用户可以控制速度、暂停、单步执行，同时右侧同步高亮对应的 C++/Java/Python 源码行，帮助理解每一步操作对应的代码逻辑。

#### 2. 结构化教程体系

教程按"绪论 → 线性表 → 树 → 图 → 查找 → 排序"六大章节组织，覆盖数据结构课程的完整知识体系。每个知识点标注了难度和预计学习时长，配合侧边栏导航，形成完整的学习路径。

#### 3. 在线编程练习与判题

练习模块分为三个标签页：

- **预设题库**：内置了基础、进阶、模式匹配、数据结构等分类的编程题，支持按难度和分类筛选，有智能推荐下一题的功能。
- **AI 出题**：接入大模型 API，根据用户选择的知识点和难度实时生成编程题或填空题，题目包含多语言代码模板、测试用例、提示和解析。
- **Vibe Coding Lab**：这是一个提示词工程训练模块，下面单独介绍。

用户编写的代码提交到判题服务后，会逐个测试用例运行，返回通过/失败、实际输出、运行时间等详细结果。判题完成后还会触发 AI 复审，对代码质量和错误原因给出智能反馈。

#### 4. Vibe Coding Lab（提示词工程训练）

这是灵码的一个特色模块，分为五个赛道：前端开发、后端开发、调试、重构、代码审查。

工作流程是：系统根据用户水平生成一个场景化挑战（比如"为一个电商后台设计分页查询接口"），用户需要撰写一段高质量的提示词来解决这个挑战，然后 AI 从目标清晰度、边界约束、验证设计、输出格式四个维度打分评估，给出优势、不足和改写示例。

前端赛道还有一个 Live Build 模式——用户通过多轮对话式提示词，让 AI 逐步生成 HTML/CSS/JS 代码，实时在 iframe 沙箱中预览渲染效果，体验"用自然语言编程"的完整流程。

系统会记录用户的历史表现，生成个人能力画像，推荐薄弱维度进行针对性训练。

#### 5. AI 思维导图

用户可以输入一个主题关键词、粘贴一段文档 URL、或上传文件，AI 会自动生成结构化的思维导图。支持节点的增删改、拖拽排序、展开折叠，还能对单个节点调用 AI 展开补充内容。思维导图支持本地存储和云端同步，可以导出为 JSON。

#### 6. 用户系统与学习追踪

平台有完整的注册登录系统，用户可以设置技能水平和目标编程语言。Dashboard 页面展示已完成课程数、练习数、连续学习天数等统计数据，系统根据学习历史做个性化推荐。

### 四、项目亮点（约1分钟）

1. **全链路覆盖**：从教程学习、可视化理解、编程练习到 AI 辅助反馈，形成完整的学习闭环。
2. **多模型适配**：AI 服务层抽象了统一接口，支持 OpenAI、DeepSeek、智谱、ModelScope 等多家大模型供应商，通过代理层转发，前端不暴露 API 密钥。
3. **安全沙箱判题**：判题容器做了只读文件系统、capability drop、PID 限制、tmpfs 隔离等多层安全防护。
4. **Vibe Coding 训练**：将提示词工程作为一项可量化、可训练的能力，这在同类学习平台中比较少见。
5. **工程化部署**：Docker Compose 三服务编排，paramiko 自动化部署脚本，健康检查验证，具备完整的生产级部署流程。
6. **国际化与无障碍**：支持中英文双语、深色/浅色主题、低动效模式（prefers-reduced-motion），关注用户体验细节。

### 附：可能的提问与准备

**Q: 为什么选择前后端分离而不是 SSR？**
A: 本项目以交互式可视化和客户端动画为核心，CSR 更适合频繁的 DOM 操作和状态管理。教程内容相对固定，SEO 需求不强，前后端分离也便于判题服务和 AI 代理的独立扩展。

**Q: 判题服务的安全性如何保证？**
A: 判题容器以 read_only 文件系统运行，仅 tmpfs 可写；drop 全部 Linux capabilities，禁止提权（no-new-privileges）；限制最大 PID 数为 256；每次判题有 5 秒超时和 64KB 输出上限；并发任务数也做了限制。

**Q: AI 出题的质量如何保证？**
A: 通过精心设计的 prompt 模板约束输出格式（JSON schema），后端对返回结果做字段校验和归一化处理。生成的题目包含多语言代码模板、测试用例、提示和解析，用户提交后还有判题+AI 复审双重验证。

**Q: Vibe Coding Lab 的评分机制是怎样的？**
A: AI 从四个维度评分：目标清晰度（goal_clarity）、边界约束（boundary_constraints）、验证设计（verification_design）、输出格式（output_format），每个维度独立打分，汇总为总分。系统记录历史成绩，生成能力画像，推荐薄弱维度训练。

**Q: 项目的并发能力如何？**
A: 前端是纯静态资源由 Nginx 托管，并发能力取决于服务器带宽。AI 请求通过 FastAPI 异步转发到第三方 API，不占用本地计算资源。判题服务是计算密集型的瓶颈，目前限制最大并发任务数为 2，可通过水平扩展判题容器来提升。
